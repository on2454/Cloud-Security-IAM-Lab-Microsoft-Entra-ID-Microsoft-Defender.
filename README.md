# Cloud Security & IAM Lab – Microsoft Entra ID & Microsoft Defender

## Project Overview

This hands-on cloud security lab focused on Identity and Access Management (IAM), administrative account security, and Microsoft Secure Score remediation using Microsoft Entra ID and Microsoft Defender.

The primary objective was to gain practical experience with identity security controls, privileged account management, multifactor authentication (MFA), authentication monitoring, and cloud security posture assessment.

---

## Technologies Used

* Microsoft Entra ID
* Microsoft Defender XDR
* Microsoft Defender Secure Score
* Microsoft 365 Admin Center
* Microsoft Authenticator
* Identity & Access Management (IAM)

---

## Project Activities

### 1. Administrative Account Configuration

* Created a dedicated Backup Administrator account
* Reviewed administrative account security requirements
* Configured authentication methods for secure access
* Validated account functionality and administrative permissions

### 2. Role-Based Access Control (RBAC)

* Assigned the Global Administrator role to the backup admin account
* Reviewed Microsoft's least-privilege recommendations
* Analyzed risks associated with permanent privileged access
* Explored role separation and administrative security best practices

### 3. Multifactor Authentication (MFA)

* Configured Microsoft Authenticator for secure authentication
* Verified MFA registration and functionality
* Reviewed available authentication methods:

  * Email
  * Phone Number
  * Microsoft Authenticator

### 4. Secure Score Assessment

Reviewed Microsoft Defender Secure Score recommendations to evaluate and improve the organization's security posture.

Implemented and validated recommendations related to:

* MFA for administrative accounts
* MFA for users
* Identity security best practices
* Authentication security improvements

**Current Secure Score:** 91.07%

### 5. Sign-In Log Investigation

Investigated Microsoft Entra ID sign-in logs to monitor authentication activity and validate security controls.

Analyzed:

* Authentication status
* MFA requirements and results
* Source IP addresses
* Applications accessed
* Session information
* Authentication details

### 6. Identity Monitoring

Monitored authentication activity for administrative accounts and validated successful sign-in events.

Key observations:

* MFA enforcement was functioning correctly
* Successful authentication events were recorded
* Token-based MFA validation was observed
* Administrative account activity was successfully monitored
